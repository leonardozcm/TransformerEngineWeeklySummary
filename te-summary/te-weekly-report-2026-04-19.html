新增 Issues 7

  • #2899 [BUG] CrossEntropyFunction.forward() modifies input in-place without ctx.mark_dirty(), causing GPU memory leak
    用户 shen-jiabin 报告：CrossEntropyFunction 通过 Triton kernel 原地修改输入但未调用 ctx.mark_dirty(inp)，导致 PyTorch C++ 层永久持有输入 tensor 引用，每次 micro-batch 泄漏 ~200-600 MiB 显存。使用梯度累积时几步后即 OOM。
    追加报告：同一作者还发现 OperationFuser._do_not_clear 在 context_parallel_size > 1 时设置的 flag 在 backward 完成后不被清除，额外泄漏 ~0.5 GiB/step。
    影响：所有通过 parallel_cross_entropy() 或 CrossEntropyFunction.apply() 使用 TE cross entropy 的训练，尤其是梯度累积场景。
    Bug Community by shen-jiabin · 04-17
  • #2897 [PyTorch] Reduce CPU overhead in grouped MLP block
    NVIDIA 内部用户 timmoon10 (Tim Moon) 提交性能跟踪 issue：GB200 上 64 experts + hidden_size=128 配置下，forward ~1.2ms、backward ~2.1ms。详细 profiling 定位了 wgrad tensor 分配（~350us）、get_device_pointer_for_data_and_scales（>100us）、clear_tensor_data（~200us）、weight quantizer 初始化（~90us）等瓶颈。
    Performance by timmoon10 · 04-16 · NVIDIA 内部追踪
  • #2896 transformer-engine-cu12 PyPI wheels have incorrect WHEEL tag

    NVIDIA 回复：ptrendx 已 tag ksivaman 跟进。

    Feature Request JAX by tdophung · 04-16
