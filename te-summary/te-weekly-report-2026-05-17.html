FeaturePerf 重要特性 & 性能优化 7

  • #2975 [JAX] Size autotuned Triton grids per config
    修复 Triton autotuned 路径中所有 BLOCK_SIZE 配置共用同一固定 grid 的问题。此前 grid 按最小 BLOCK_SIZE 固定，导致大配置过度启动。改为 grid 接受 callable(meta)->tuple 按配置评估。GB200 上 sort_chunks (524k tokens, hidden=4096, fp32) 从 22.6ms 降至 7.4ms，提速 3.06 倍。
    3x Speedup JAX Triton by tdophung · merged 05-14
  • #2900 [PyTorch] Expose function to bulk-allocate tensors backed by the same buffer
    泛化 split-quantize 中的 bulk allocation 方案：分配单个大 buffer，通过 at::from_blob 创建 tensor subview。在 GroupedLinear 中按 expert 分配 tensor 时的 CPU overhead 显著降低。增量推进 #2897（CPU overhead 追踪 issue）。
    CPU Overhead MoE by timmoon10 · merged 05-12
  • #2958 [Common] Use specialized unfused MXFP8 cast kernels by default
    将快速 unfused MXFP8 cast kernel 设为默认启用。此前这些 kernel 被环境变量 gate 住需要显式开启。这一变更使 MXFP8 的 cast-only 路径直接走优化后的专用 kernel，无需用户手动配置。上周作为 Open PR 提出，本周成功合入。
    Performance MXFP8 by Oleg-Goncharov · merged 05-12
  • #2957 [PyTorch] CPU overhead optimizations for te autocast
    优化 te-autocast 的 CPU 开销，在 Grace 系统上效果显著。具体措施：(1) 缓存 recipe 字符串表示以加速 autocast key 计算；(2) 直接访问缓存的表示比经过 str(recipe) 快约 40ns。上周作为 Open PR 提出，本周合入。
    CPU Overhead Grace/Blackwell by vthumbe1503 · merged 05-12
  • #2620 [PyTorch] Introduce quantizer roles
    引入 QuantizerRole dataclass（含 module_type、tensor_type、name 字段），提供细粒度的 per-module 量化控制 API。允许针对特定 transformer layer 中的特定 LayerNormLinear 设置不同的量化策略，为实际部署中"关键层少量化、非关键层激进量化"的混合精度策略提供支撑。
    Quantization API by negvet · merged 05-11
  • #2793 Avoid CPU offload wait_event for validation
    在 validation 阶段跳过 CPU offload 的 wait_event 同步，减少不必要的 CPU-GPU 同步开销。对使用 CPU offloading 的训练流程在 validation 阶段有直接的延迟改善。
    CPU Overhead Offload by vasunvidia · merged 05-12
  • #2666 [PyTorch] Replace the make_empty implementation to use C++ implementation
    将 QuantizedTensor 的创建路径统一为 C++ 实现的 create_tensor，替代原有 Python 层的 make_empty。减少跨语言调用开销，统一内存分配路径。
    Perf QuantizedTensor by ptrendx · merged 05-13

  • #2842 [Core] Report CUDA versions when NVRTC compilation fails
