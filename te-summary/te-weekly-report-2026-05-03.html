  • #2865 Optimizations for MXFP8/NVFP4 dequantize kernels
    多项 MXFP8/NVFP4 dequantize kernel 优化：(1) 处理空 tensor 以兼容 CUDA Graph；(2) 为 NVFP4 dequantize 添加 swizzled scale 支持，复用已有的 MXFP8 swizzle index 计算逻辑；(3) 新增 NVFP4 和 MXFP8 dequantization 的 C++ 单元测试（含 swizzled scale 变体）；(4) 修复测试基础设施中 to_cpu() 和 set_scale() 在 NVTE_NVFP4_1D_SCALING 模式下的 amax/scale 同步问题。
    Optimization MXFP8 NVFP4 by YigongQin · merged 05-02
  • #2522 [JAX] Calculate seqlens and offsets in O(T) space instead of O(T*T) for THD sequences
    将 TE JAX attention 中 THD seqlens/offsets 计算的空间复杂度从 O(T²) 降至 O(T*max_segment_per_seq)。此前实现会具化完整的 mask 矩阵，在长序列场景下容易导致 OOM。新方案在降低内存的同时也带来了一些性能提升。标记为 2.15.0。
    JAX Memory O(T) 2.15.0 by KshitijLakhani · merged 05-01
  • #2914 Variable Grouped Swizzle
    支持可变 shape 的 grouped swizzle kernel，解决此前仅支持 uniform shape 的限制。修复 #2451。Breaking Change — 因为变更了 grouped swizzle 的接口签名。
    MoE Swizzle by int-smart · merged 04-30

新增 Issues 3

  • #2956 [BUG] NVFP4BlockScaling forward pass fails on RTX 5080 (sm_120)
    用户 osubotin 报告：RTX 5080 (消费级 Blackwell, sm_120) 上 is_nvfp4_available() 返回 True，但实际 NVFP4 forward kernel 在大多数生产尺寸下报 CUDA Error: invalid argument。根因与 PyTorch 上游问题 pytorch/pytorch#172807（gencode 自动检测在 Blackwell 消费级 arch 上丢失 a 后缀）有关。sm_100 的 TMEM+UMMA 在 row_cast_col_hadamard_transform_cast_fusion.cu 中被硬编码，导致 sm_120 不兼容。
    影响：所有消费级 Blackwell GPU 上的 NVFP4 使用场景。
    Bug Community by osubotin · 05-03
  • #2945 Return softmax aux statistics from JAX fused attention bindings
    用户 zeryx 提出功能请求：JAX fused attention binding 在 forward pass 中丢弃了 softmax_aux 统计信息，而 PyTorch binding 正确返回了这些信息（作为 aux_ctx_tensors）。这造成了两个前端之间的功能差距，阻止 JAX 用户访问中间 softmax 状态（用于检查、日志记录或自定义 backward pass）。
    影响：JAX 前端的功能完整性。目前无评论回复。
    Feature Request JAX Community by zeryx · 04-30
  • #2939 GroupedLinear performance on Mixtral-8x7B at EP=2
    用户 faradawn 报告：在 8× B300、BF16、EP=2 配置下，transformer_engine.pytorch.GroupedLinear 比 Python 循环调用 F.linear 慢约 2 倍。Batch=1 时 loop 240ms vs GroupedLinear 449ms (1.87×)，Batch=4 时 242ms vs 511ms (2.11×)。
    作者跟进：已有对应 PR #2923 (Make TE Sequential Grouped linear Op CUDA graphable) 在推进中。
    Performance MoE Community by faradawn · 04-28
