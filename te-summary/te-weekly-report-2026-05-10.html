  • #2959 [PyTorch/Common] Remove legacy FP8DS implementation

    Breaking 2.16.0 by cyanguwa · merged 05-07
  • #2949 [All] Remove legacy max512 backend
    移除 fused attention 的 f16_max512 子后端（FP16/BF16, max_seq_len≤512, head_dim=64, MHA only）。该后端基于旧版 cuDNN 接口，即将被 cudnn-frontend 移除。所有 BF16/FP16 attention 将统一走 f16_arbitrary 路径。标记为 2.16.0。
    Breaking Attention 2.16.0 by cyanguwa · merged 05-07
  • #2963 [Common] Always define cuBLASMp comm GEMM API
    使 cuBLASMp 相关的 TE API 无条件定义。NVTE_WITH_CUBLASMP 选项现在控制用户获得真实实现还是 stub（运行时报错），避免了编译时条件导致的 API 缺失问题。标记为 2.16.0。

FeaturePerf 特性 & 性能优化 5

  • #2972 Implement 4over6 NVFP4 recipe
    本周亮点 Open PR。实现来自 MIT HAN Lab 的 4over6 论文的 NVFP4 量化方案。通过 per-block 选择 map-to-4 vs map-to-6 候选集，提升 NVFP4 1D quantization 的精度。当前要求禁用 RHT、stochastic rounding 和 2D quantization。同时支持 per-tensor scaling 和 row-scaling 模式。已有配套 FlashInfer PR (flashinfer#3264)。
    NVFP4 4over6 by zianglih · 05-09
  • #2970 [JAX][PyT] Tighten SWA checks in DPA, MHA and other APIs before passing onto cuDNN fused attn & unfused attn backends
    统一 JAX 和 PyTorch 的 sliding window attention (SWA) 参数校验逻辑。JAX 侧仿照 PyTorch 的 check_set_window_size() 在上层对 window_size 进行规范化，使得传递到后端的值集合更加受限和可预期。提升 SWA 的跨前端一致性。
    Attention JAX reviewer: cyanguwa by KshitijLakhani · 05-08
  • #2958 [Common] Use specialized unfused MXFP8 cast kernels by default
    将快速 unfused MXFP8 cast kernel 设为默认启用。此前这些 kernel 被环境变量 gate 住，需要显式开启才能使用。这一变更使得 MXFP8 的 cast-only 路径直接走优化后的专用 kernel。
    Performance MXFP8 by Oleg-Goncharov · 05-05
  • #2957 CPU overhead optimizations for te autocast
    优化 te-autocast 的 CPU 开销，在 Grace 系统上效果显著。具体措施：(1) 缓存 recipe 字符串表示以加速 autocast key 计算；(2) 直接访问缓存的表示比经过 str(recipe) 快约 40ns。在 GB200 上取得可观的延迟降低。
    CPU Overhead Grace/Blackwell by vthumbe1503 · 05-04
  • #2960 Draft: Extended Tensor Parallelism
    引入扩展 Tensor Parallelism 的初步实现。目前为 Draft 状态，详细描述尚未填写。预计将为 TE 提供更灵活的 TP 策略支持。
    Parallelism Draft by jiemingz · 05-05
